# Prompt 02 - Build Expedition Board UI

Context files to use:
- `game_overview.md`
- `current_milestone.md`
- `technical_decisions.md`
- `coding_standards.md`
- `content_schema.md`

Task:
Implement the Expedition Board screen and its controller.

Requirements:
- Create a screen that displays 3 to 5 expedition cards generated by `ExpeditionGenerator.gd`.
- Each expedition card must clearly show:
  - display name
  - duration
  - risk label
  - reward profile
  - hazard
- The player should be able to tap/select one expedition and move to a dispatch flow.
- The board UI should be mobile-friendly and readable with placeholder styling.

Constraints:
- Godot 4, GDScript only.
- Keep the layout simple.
- Do not implement advanced art, filters, specialist recommendations, or refresh economy yet.
- Do not redesign unrelated screens.

Deliverables:
- `scenes/expedition_board/ExpeditionBoard.tscn`
- `scripts/ui/ExpeditionBoardController.gd`
- reusable expedition card scene if helpful, e.g. `scenes/components/ExpeditionCard.tscn`

Acceptance criteria:
- The screen shows valid generated expeditions.
- The player can select one expedition.
- The selected expedition data can be passed into the next screen or modal cleanly.
